## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | **Start Simulation** |
| Empty state heading | No area selected |
| Empty state body | Draw a rectangle on the map to define the plantation area, then press Start Simulation. |
| Error state: missing Cesium token | Cesium token required — Set `VITE_CESIUM_ION_TOKEN` in `.env.local` to enable 3D terrain and imagery. |
| Error state: terrain failure | Terrain loading failed — Scene will render with flat elevation. Generation continues normally. |
| Error state: generation delay | Generating plantation... — Large selections may take a few seconds. |
| Destructive confirmation | Reset Selection: This will clear the current area and stop any running simulation. Continue? |

**Interaction Copy:**

| Interaction | Label/Message |
|-------------|---------------|
| Selection in progress | Selection: {area} km² — Release to confirm |
| Selection confirmed | Area confirmed: {area} km² — Press Start to begin |
| Simulation running | Scanning plantation... — {coverage}% complete |
| Simulation complete | Scan complete — {trees} trees analyzed |

**Phase-Specific Notes:**
- Selection area should display in square kilometers with 2 decimal precision
- Coverage percentage should update as sweep path progresses
- Tree count should reflect generated plantation size
- All messages should feel credible for a plantation monitoring demo, not a debug UI

## Component Inventory

Phase 2 introduces these new UI components:

### 1. Cesium Map Panel (replaces ArcGIS placeholder)

**Purpose:** Display Cesium globe with area selection capability

**States:**
- **Loading:** "Initializing Cesium..." with spinner
- **No token:** Error state with token setup instructions (see Copywriting Contract)
- **Idle:** Map visible, no selection active
- **Selecting:** User is dragging rectangle, shows live area calculation
- **Confirmed:** Rectangle locked, Start Simulation enabled
- **Running:** Rectangle remains visible, drone path overlays appear in later phases

**Visual Contract:**
- Cesium viewer fills `#map-panel` container (existing `.panel__surface--map`)
- Selection rectangle: 2px solid `#38bdf8` stroke, `rgba(56, 189, 248, 0.15)` fill
- Selection handles: 8px white circles at corners (touch-friendly)
- Area label: floating badge near top-right corner of rectangle, dark background with white text

**Interaction:**
- Click-drag on map to create/resize rectangle
- Drag rectangle body to reposition without resizing
- Drag corner handles to resize
- Double-click rectangle to confirm and lock
- Escape key or Cancel button to clear selection

### 2. Selection Status Badge

**Purpose:** Display confirmed area size and simulation readiness

**Location:** Panel footer below map (existing `.panel__footer`)

**Visual Contract:**
- Badge component using existing `.badge` class
- When no selection: muted badge "No area selected"
- When selected: accent badge "Area: {X.XX} km² — Ready"
- When running: success badge "Scanning: {coverage}% complete"

### 3. Start Simulation Button

**Purpose:** Explicit trigger for simulation after area confirmation

**Location:** Shell header controls (existing `.shell__controls`)

**States:**
- **Disabled:** No area selected, button grayed out with `.button--secondary` style, label "Start Simulation"
- **Ready:** Area confirmed, button uses primary gradient style (existing default button)
- **Running:** Button replaced by "Stop" button with `.button--secondary` and "Reset" button with `.button--ghost`

**Visual Contract:**
- Uses existing button system from `main.css`
- Disabled state: `opacity: 0.5`, `cursor: not-allowed`
- Active state: existing gradient `linear-gradient(135deg, #38bdf8, #22c55e)`

### 4. 3D Scene Panel (existing, enhanced)

**Purpose:** Display Three.js plantation scene with terrain-aware drone

**States:**
- **Empty:** "No plantation generated" placeholder (before selection)
- **Generating:** "Generating plantation..." with spinner (after Start pressed)
- **Ready:** Scene rendered, drone at starting position
- **Running:** Drone animated along sweep path
- **Error:** "Scene generation failed" fallback

**Visual Contract:**
- Existing container `#scene-panel` (.panel__surface--scene)
- Generation spinner: centered, cyan accent color
- Fallback state: existing panel footer with error message

**Interaction:**
- No user controls in Phase 2 (orbit/pan added in later phases)
- Scene auto-rebuilds when new area is confirmed

### 5. Generation Loading Overlay

**Purpose:** Indicate procedural plantation generation in progress

**Visual Contract:**
- Full-height overlay on 3D scene panel
- Semi-transparent dark background `rgba(2, 8, 23, 0.9)`
- Centered spinner with "Generating plantation..." text
- Estimated time indicator for large selections (>2 km²)

**Timing:**
- Appears when Start Simulation pressed
- Remains visible until world generation completes
- Fades out over 300ms when scene is ready

## Interaction Flows

### Flow 1: Area Selection to Simulation Start

```
1. [Idle] User sees Cesium map, no selection
2. [Action] User clicks and drags on map
3. [Selecting] Rectangle appears, area label updates live
4. [Action] User releases mouse
5. [Confirmed] Rectangle locks, badge shows "Area: X.XX km² — Ready"
6. [Action] User clicks "Start Simulation"
7. [Generating] Loading overlay on 3D panel, "Generating plantation..."
8. [Ready] Scene renders with plantation, drone at start position
9. [Running] Drone begins sweep path, coverage updates
```

### Flow 2: Terrain Failure Graceful Degradation

```
1. [Selection confirmed] User presses Start Simulation
2. [Terrain request] System attempts to sample Cesium terrain
3. [Failure detected] Terrain service times out or returns error
4. [Fallback] Scene renders with flat elevation (no crash)
5. [User notification] Small badge in scene footer: "Terrain: unavailable (flat elevation)"
6. [Continue] Simulation proceeds normally with flat terrain
```

### Flow 3: Missing Cesium Token

```
1. [Load] App initializes, checks for VITE_CESIUM_ION_TOKEN
2. [Missing] Token not found in environment config
3. [Fallback UI] Map panel shows error state with setup instructions
4. [User reads] "Cesium token required — Set VITE_CESIUM_ION_TOKEN..."
5. [Graceful] Rest of app remains functional, 3D panel can still render local geometry
```

## Phase-Specific Visual Principles

### Plantation Demo Credibility
- **Orderly rows:** Generated plantation should read as organized palm plantation rows, not random forest
- **Natural variation:** Slight per-tree position offsets (±0.5m) to avoid perfect grid appearance
- **Terrain following:** Drone altitude should vary with terrain, not stay at fixed height
- **Realistic scale:** Tree spacing 8-10m, drone altitude 12-20m above ground

### Deliberate Interaction Flow
- **Explicit confirmation:** Selection requires double-click or explicit confirm action (no auto-start)
- **Visual feedback:** Every user action has immediate visual response (selection rectangle, button states)
- **State clarity:** User always knows if system is idle/selecting/ready/running

### Performance-Aware UX
- **Loading states:** Always show loading indicators for async operations (Cesium init, terrain sampling, generation)
- **Graceful degradation:** Missing tokens or failed terrain requests never crash the app
- **Error recovery:** All error states include clear next steps for user
