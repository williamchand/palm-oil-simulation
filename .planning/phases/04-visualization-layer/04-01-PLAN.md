**Create `src/three/helpers/heatmapOverlay.js`** — per D-21, D-24, D-25:

Export `createHeatmapOverlay(plantationData)` factory returning `{ mesh, reveal(waypointX, waypointY, trees, anomalies), dispose() }`.

Implementation:
1. Create an offscreen `<canvas>` sized to the plantation grid. Calculate pixel dimensions: `cols` × `rows` pixels (or scaled up for resolution, e.g., `cols * 2` × `rows * 2` — cap at 512×512 for performance). Fill canvas entirely with rgba(0, 0, 0, 1) — this is the fog-of-war dark state (D-24).
2. Create a `CanvasTexture` from this canvas. Create a `PlaneGeometry` matching the ground plane dimensions (`cols * spacing * 0.1 * 1.2` wide, `rows * spacing * 0.1 * 1.2` tall — matching sceneBuilder.js ground sizing). Create a `Mesh` with `MeshBasicMaterial({ map: texture, transparent: true, opacity: 0.85, depthWrite: false })`. Position identically to the sceneBuilder ground: `rotation.x = -Math.PI/2`, `position.set(widthUnits/2, 0.01, heightUnits/2)` (0.01 Y offset to float above ground and avoid z-fighting). Name it `'heatmapOverlay'`.
3. `reveal(waypointX, waypointY, trees, anomalies)` method:
   - **Throttle** per D-25: track `lastUpdateTime`. If less than 1000ms since last update, skip (return early). Update `lastUpdateTime` on actual paint.
   - Convert waypoint position (meters) to canvas pixel coordinates: `pixelX = (waypointX / (cols * spacing)) * canvasWidth`, `pixelY = (waypointY / (rows * spacing)) * canvasHeight`.
   - Calculate reveal radius in pixels: use 30m scan radius → `radiusPx = (30 / (cols * spacing)) * canvasWidth`.
   - Use `ctx.globalCompositeOperation = 'source-over'` and paint a radial gradient circle at (pixelX, pixelY) with `radiusPx`:
     - For each nearby tree (within 30m of waypoint), compute its pixel position and paint a small circle (radius ~3-5px):
       - Color based on health + ripeness per D-21: `health < 0.4` → red (#ef4444), `health < 0.7` → yellow (#eab308) with ripeness modulating intensity (alpha = 0.5 + ripeness * 0.5), `health >= 0.7` → green (#22c55e) with ripeness modulating brightness.
     - Paint the base reveal area (circle around waypoint) with a translucent dark-green base (#1a2f1a at alpha 0.6) to "lift" the fog.
   - After painting, set `texture.needsUpdate = true`.
   - If anomalies array has entries for this area, paint additional red highlight circles for each anomaly position.
4. `dispose()`: dispose geometry, material, texture.

Also export a pure function `getHealthColor(health, ripeness)` for testability:
- `health < 0.4` → `{ r: 239, g: 68, b: 68, a: 0.5 + ripeness * 0.5 }` (red/diseased)
- `health < 0.7` → `{ r: 234, g: 179, b: 8, a: 0.5 + ripeness * 0.5 }` (yellow/stressed)
- `health >= 0.7` → `{ r: 34, g: 197, b: 94, a: 0.5 + ripeness * 0.5 }` (green/healthy)

**Create `src/three/helpers/pathTrail.js`** — per D-22, D-23:

Export `createPathTrail()` factory returning `{ group, addPoint(x, z, altitude, action, isAnomaly), dispose() }`.

Implementation:
1. Create a `Group` named `'pathTrail'`.
2. For the **line trail** (D-22): Use `BufferGeometry` with a pre-allocated `Float32Array` position buffer (max 10000 points × 3 floats) and a `Float32Array` color buffer (max 10000 × 3 floats). Set `drawRange` count to 0. Create a `Line` (from three) with `LineBasicMaterial({ vertexColors: true })`. Add to group.
3. For **scan point markers** (D-23): Create `InstancedMesh` with `SphereGeometry(0.15, 8, 6)` and `MeshBasicMaterial({ color: '#38bdf8' })`, max count 2000. Track `markerCount = 0`. Add to group.
4. `addPoint(x, z, altitude, action, isAnomaly)` method:
   - Convert to scene units: `sx = x * 0.1`, `sy = altitude * 0.1`, `sz = z * 0.1`.
   - Append position (sx, sy, sz) to the positions buffer at current point index.
   - Determine color per D-22: if `isAnomaly` → red (1, 0.27, 0.27), else if `action === 'scan'` → blue (0.22, 0.74, 0.97), else if `action === 'turn'` → gray (0.39, 0.45, 0.53), else → gray.
   - Append color to colors buffer. Increment point count. Update `geometry.attributes.position.needsUpdate = true`, `geometry.attributes.color.needsUpdate = true`, `geometry.setDrawRange(0, pointCount)`.
   - If `action === 'scan'`: place an instanced sphere marker at (sx, sy, sz). Use `Object3D` dummy, set position, `updateMatrix()`, `instancedMesh.setMatrixAt(markerCount, dummy.matrix)`. If `isAnomaly`, set color to red via `instancedMesh.setColorAt(markerCount, redColor)`. Increment `markerCount`. Update `instancedMesh.instanceMatrix.needsUpdate = true`.
5. `dispose()`: dispose all geometries and materials in group.

Also export a pure function `getSegmentColor(action, isAnomaly)` for testability:
- `isAnomaly` → `{ r: 1, g: 0.27, b: 0.27 }` (red #ef4444)
- `action === 'scan'` → `{ r: 0.22, g: 0.74, b: 0.97 }` (blue #38bdf8)
- default → `{ r: 0.39, g: 0.45, b: 0.53 }` (gray #64748b)
