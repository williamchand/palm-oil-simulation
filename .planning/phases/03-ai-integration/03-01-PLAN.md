`buildSystemPrompt()` — returns a multi-line string (per D-18):
```
You are an AI-powered agricultural survey drone scanning a palm oil plantation.
Your task is to analyze environmental perception data and make autonomous decisions.

You MUST respond with valid JSON only. No other text.

Response format:
{
  "type": "adjust_priority" | "flag_anomaly" | "modify_altitude",
  "reasoning": "Brief explanation of your decision",
  "confidence": 0.0 to 1.0,
  "parameters": { ... decision-specific parameters ... }
}

Decision types:
1. "adjust_priority" — Reorder upcoming scan waypoints. Parameters: { "priorityIndices": [int, ...] }
2. "flag_anomaly" — Flag detected anomaly. Parameters: { "anomalyType": "low_health"|"pest_cluster"|"ripeness_gradient", "anomalyDescription": "..." }
3. "modify_altitude" — Change scan altitude. Parameters: { "altitudeChange": float (meters, positive=up, negative=down), range -5 to +10 }

Rules:
- Prioritize flagging anomalies when you detect clusters of unhealthy trees (health < 0.3)
- Suggest altitude changes when ripeness data is ambiguous (many trees with ripeness 0.4-0.6)
- Adjust priority when you detect a gradient suggesting nearby high-value areas
- If conditions are normal, use "modify_altitude" with altitudeChange: 0 and reasoning explaining normal conditions
```

`buildUserPrompt(perception)` — returns:

Current perception at waypoint ${perception.dronePosition.waypointIndex}:

Analyze this data and decide your next action.

`buildRequestBody(perception)` — returns (per D-17, D-18):

`parseAiResponse(rawText)` — parse raw LLM text to AiDecision:
- Try `JSON.parse(rawText)`. If that fails, try to extract JSON from the text using regex `/\{[\s\S]*\}/` and parse that.
- Validate required fields: `type` must be one of `['adjust_priority', 'flag_anomaly', 'modify_altitude']`, `reasoning` must be a string, `confidence` must be a number 0-1.
- If `parameters` is missing, default to `{}`.
- If validation fails, return `null`.
- If valid, return the `AiDecision` object.

`export function createGemmaClient(config)`:
- Extract `apiUrl = config.gemmaApiUrl`, `apiKey = config.gemmaApiKey`
- Determine mode: `const mode = apiUrl ? 'api' : 'mock'`
- Return object with: `{ infer, getMode, getBootSequence }`

`getMode()` — returns `'api'` or `'mock'`

`getBootSequence()` — keep backward compat for Plan 03-03 transition. Return the same static array as current implementation. This will be removed in Plan 03-03.

`async infer(perception)` — the main method:
- Record `startTime = Date.now()`
- If mode === 'mock': call `generateMockDecision(perception)`, return `{ success: true, decision, rawResponse: JSON.stringify(decision), latencyMs: Date.now() - startTime, source: 'mock' }`
- If mode === 'api':
  - Build request body: `const body = buildRequestBody(perception)`
  - Try `fetch(apiUrl, { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': apiKey ? 'Bearer ' + apiKey : undefined }, body: JSON.stringify(body), signal: AbortSignal.timeout(5000) })`
  - Filter out undefined Authorization header
  - Parse response: `const data = await response.json()`, extract text from `data.choices?.[0]?.message?.content` or `data.text` or `data.response` (handle various API response shapes)
  - Call `parseAiResponse(text)` to get decision
  - If decision is null, return `{ success: false, decision: null, rawResponse: text, latencyMs, source: 'api' }`
  - If decision valid, return `{ success: true, decision, rawResponse: text, latencyMs, source: 'api' }`
  - On any error (fetch fail, timeout, parse error): return `{ success: false, decision: null, rawResponse: error.message, latencyMs: Date.now() - startTime, source: 'fallback' }` — per D-14 graceful degradation

`function generateMockDecision(perception)` (internal, not exported):
- Analyze perception data to generate contextually relevant mock decisions
- If any `nearbyTree.health < 0.3` exists: return `flag_anomaly` with `anomalyType: 'low_health'`, describe which trees
- Else if average ripeness of nearby trees is between 0.4-0.6: return `modify_altitude` with `altitudeChange: -3` (go lower for better scan)
- Else if `coverageStats.percentComplete > 70`: return `adjust_priority` with empty priorityIndices (maintain course)
- Default: return `modify_altitude` with `altitudeChange: 0`, reasoning: "Conditions normal, maintaining current altitude"
- Always include `confidence: 0.75` for mock, `reasoning` describing the logic
