**graceful-degradation.test.js** — Test AI failure handling across the pipeline:

1. Create a failing gemmaClient mock: `infer: async () => { throw new Error('Network timeout'); }`, getMode: () => 'mock'.

2. Test "AI failure does not crash AI waypoint loop":
   - Create simState, selection, plantation, route (same helpers as e2e-flow)
   - Create aiLoop with failing client
   - Iterate through 3 scan waypoints via `onWaypoint()` — all should resolve (not throw)
   - Verify `aiLoop.getStats().failedInferences === 3`
   - Verify `aiLoop.getStats().successfulInferences === 0`

3. Test "reasoning log records fallback entries on failure":
   - After failing inferences, check `simState.getState().reasoningLog`
   - Each entry should have `source: 'fallback'` and `decision: null`

4. Test "aiStatus set to degraded on failure":
   - After a failing inference, verify `simState.getState().aiStatus === 'degraded'`

5. Test "mixed success/failure — partial results kept":
   - Create a gemma client that succeeds on first call, fails on second
   - Run 2 scan waypoints
   - Verify stats: 1 success, 1 failure
   - Verify reasoning log has 2 entries (one with decision, one without)

6. Test "null decision from AI is a no-op in applyDecision":
   - `applyDecision(null, route, 0)` → `{ applied: false }`

7. Test "invalid decision type is a no-op":
   - `applyDecision({ type: 'invalid_type', reasoning: '', confidence: 0, parameters: {} }, route, 0)` → `{ applied: false }`

**performance-snapshot.test.js** — Measure integration overhead:

1. Test "plantation generation completes in under 200ms for 1km² area":

   - Using mock gemma client (instant responses)
   - Time: generatePlantation + createSweepPath + 5x onWaypoint (await each)
   - Assert total < 500ms

All performance thresholds are generous — they catch regressions, not micro-optimize. Use `performance.now()` for timing.
